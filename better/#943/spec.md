# 인출설계 (Withdrawal Plan) — #943

## 개요

연결된 마이데이터 자산을 기반으로 사용자의 은퇴 후 자산 인출 전략을 설계하는 기능.
FA(자산관리사)에게 설문 결과를 전달하고, FA가 맞춤 인출 전략을 제안하는 흐름으로 연결된다.

---

## 진입 조건

- **위치**: 마이데이터 자산동기화 완료 화면 (AssetSyncResult)
- **조건**: 자산동기화 **성공** + 사용자 만 나이 **55세 초과**인 경우에만 바텀시트 자동 노출
- **노출 형태**: 하단 바텀시트 자동 팝업

### 바텀시트 선택지
| 선택 | 동작 |
|------|------|
| 네 | 인출설계 인트로 화면으로 이동 |
| 아니요 | 바텀시트 닫힘, 기존 화면(자산동기화 완료) 유지 |

---

## 전체 플로우

```
자산동기화 완료 화면
    ↓ (55세 초과 + 동기화 성공 → 바텀시트 노출)
    ↓ "네" 선택
[Intro] 인출설계 소개
    ↓ 시작하기
    ├─ IRP/DC 자산 없음 → [NoAsset] 자산 없음 안내
    └─ IRP/DC 자산 있음 → [Survey] 설문 (최대 6단계)
                              ↓ 설문 완료
                          [Confirm] 입력 내용 확인
                              ↓ FA에게 전달하기
                              ├─ T0054 이미 동의 → [Loading] 전달 중...
                              └─ T0054 미동의 → 약관 동의 바텀시트 팝업
                                                    ↓ 동의하고 전달하기
                                                [Loading] 전달 중...
                                                    ↓
                                                [Result] 완료 / 실패
```

## 화면별 상세

### 1. Intro (인출설계 소개)
- 기능 소개 문구 및 소요 시간(약 3분) 안내
- **시작하기** 버튼 탭 시:
  - 마이데이터에 연결된 IRP 또는 DC 계좌가 있는지 자동 확인
  - 있으면 → 설문으로 이동
  - 없으면 → NoAsset 화면으로 이동

---

### 2. NoAsset (자산 없음 안내)
- 퇴직연금(IRP/DC) 계좌가 마이데이터에 연결되지 않은 경우 표시
- **마이데이터 연결하기** → 마이데이터 화면으로 이동
- **나중에 하기** → 메인 홈으로 이동

---

### 3. Survey (설문, 최대 6단계)

상단에 진행률 프로그레스바 표시. 모든 단계는 이전/다음 이동 가능.

#### Step 1. 은퇴 시기
- 질문: "언제 은퇴하고 싶으신가요?"
- 입력 방식: 슬라이더 (범위: 만 40세 ~ 80세, 기본값 60세)
- 참고 정보: 대한민국 평균 은퇴 나이(49세), 희망 은퇴(65세) 표시

#### Step 2. 기대수명
- 질문: "기대수명은 얼마로 생각하시나요?"
- 입력 방식: 슬라이더 (범위: 70세 ~ 100세, 기본값 90세)
- 참고 정보: 남성 81세, 여성 87세 기대수명 표시 (출처: 통계청)

#### Step 3. 월 희망 수령액
- 질문: "은퇴 후 연금을 얼마씩 받고 싶으세요?"
- 입력 방식: 슬라이더 (범위: 120만원 ~ 1,000만원)
- 참고 정보: 최소 생활비 120만원, 적정 생활비 165만원 표시 (출처: 국민연금공단)

#### Step 4. 국민연금
- 질문: "국민연금을 수령중이신가요?"
- 선택지 2가지: 네 (현재 수령 중) / 아니요 (미수령)
- 선택과 무관하게 월 수령액 인풋 1개 표시
  - 네 선택 시: 실제 월 수령액 입력 (세전, 만원/월)
  - 아니요 선택 시: 예상 월 수령액 입력 (세전, 만원/월)
- 인풋 하단 **"월 수령액 계산기"** 버튼 → 바텀시트 오픈

  **계산기 바텀시트 내용**
  - 세전 연소득 입력 (만원)
  - 국민연금 최초 가입 시기 (연도 + 월 선택)
  - 납입 종료 예정일: 별도 박스 없이 "만 60세 기준으로 자동 적용돼요" 캡션 옆에 `YYYY-MM` 인라인 표시
  - **계산 후 적용하기** 버튼 (연소득 + 가입 시기 모두 입력 시 활성화)
    - 클릭 시: 계산 → 월 수령액 인풋에 자동 입력 → 인풋 포커싱 → 바텀시트 닫힘
    - 별도 계산 결과 표시 없음 (계산과 동시에 적용)
  - 바텀시트 닫힘(X 버튼 또는 계산 후 적용하기) 시 계산기 입력값 초기화
  - 바텀시트 푸터 닫기/확인 버튼 없음 (X 버튼으로만 닫기 가능)

  > ⚠️ 납입 종료 예정일은 사용자 생년월일 + 만 60세로 계산된 실제 년/월 표시 (은퇴 시기와 무관하게 고정)
  > ⚠️ 계산기 보조 입력값(연소득, 최초 가입 시기)은 최종 제출에 포함되지 않음. 계산된 월 수령액만 전달.

#### Step 5. 근로소득 *(DC 자산 보유자는 이 단계 자동 생략)*
- 질문: "현재 근로소득이 있나요?"
- 선택지 2가지:

  **A. 있어요**
  - 세전 연봉 입력 (만원)
  - 근속연수 입력 (년)

  **B. 없어요**
  - 추가 입력 없음

  > ⚠️ DC 자산 보유 여부는 마이데이터 연결 정보를 기반으로 자동 판단. 사용자에게 별도로 묻지 않음.

#### Step 6. 기타 정기소득
- 질문: "은퇴 후 기타 정기소득이 있거나 예정인가요?"
- 선택지 2가지:

  **A. 있어요**
  - 세전 연소득 입력 (만원/년) — 월세, 임대, 배당 등
  - 소득 시작 시기: Step 1 은퇴 시기 기준 자동 계산 (잠금 처리)
  - 소득 종료 시기: Step 2 기대수명 기준 자동 계산 (잠금 처리)

  **B. 없어요**
  - 추가 입력 없음

  > ⚠️ 소득 시작/종료 시기는 사용자 생년월일 + 해당 나이로 계산된 실제 년/월 표시

---

### 4. Confirm (입력 내용 확인)

설문에서 입력한 항목 전체를 목록으로 표시.

| 항목 | 표시 조건 |
|------|-----------|
| 은퇴 예상 시기 | 항상 표시 |
| 기대수명 | 항상 표시 |
| 월 생활비 | 항상 표시 |
| 국민연금 월 수령액 | 항상 표시 |
| 근로소득 | DC 자산 보유자는 **표시 안 함** |
| 기타 정기소득 | 항상 표시 |

- 각 항목 우측에 **수정** 버튼 → 해당 설문 단계로 이동
- 수정 완료 후 돌아오면 "**수정완료**" 버튼으로 이 화면으로 복귀
- **FA에게 전달하기** → 약관 동의 화면으로 이동

---

### 5. Terms (약관 동의 바텀시트)

별도 페이지 없이 Confirm 화면 위에 바텀시트로 표시.

- **트리거**: Confirm 화면 "FA에게 전달하기" 버튼 → T0054 미동의 시 자동 오픈
- T0054(개인정보 제3자 제공동의) 약관 항목 표시
- **동의하고 전달하기** 버튼 → 동의 API 제출 → Loading으로 이동
- 바텀시트 닫기 → Confirm 화면으로 복귀 (전달 안 함)

---

### 6. Loading (전달 중)

- FA에게 데이터 전달 중 로딩 화면
- 완료 후 Result 화면으로 자동 이동

---

### 7. Result (완료 / 실패)

**성공 시**
- 완료 메시지 표시
- "담당 FA가 맞춤 인출 전략으로 곧 연락드릴게요 (평균 1~2영업일)"
- **홈으로 돌아가기** 버튼

**실패 시**
- 오류 메시지 표시
- **다시 시도** / **홈으로 이동** 버튼

---

## 주요 체크사항

### 데이터 임시저장
- 설문 도중 앱을 나가거나 화면을 벗어나도 입력 내용이 유지됨
- FA 전달 완료 후 임시저장 데이터는 자동 삭제됨
- 다음에 다시 인출설계를 시작하면 이전 입력값을 불러와 이어서 작업 가능

### DC 자산 자동 판단
- 마이데이터에 DC(확정기여형) 퇴직연금 계좌가 연결된 경우, 근로소득 설문 단계를 자동으로 건너뜀
- DC 자산 보유자는 근로소득 정보를 수집하지 않으므로 확인 화면에서도 해당 항목이 표시되지 않음

### 자동 계산 항목
- 국민연금 납입 종료 예정일, 기타소득 시작·종료 시기는 **사용자가 직접 입력하지 않음**
- 생년월일을 기준으로 만 N세가 되는 실제 년/월을 계산하여 표시
- 잠금(🔒) 아이콘으로 자동 적용됨을 시각적으로 안내

### 인풋 자동 포커싱
- 선택지(네/아니요) 선택 후 인풋이 화면에 나타나는 시점에 첫 번째 인풋 자동 포커싱
- 수정 모드 진입 시에도 동일하게 동작
- 적용 화면:
  - Nps: isReceivingNps 선택 → npsMonthlyAmount 인풋 포커싱
  - Nps 계산기: 계산 후 적용하기 클릭 → 계산기 닫힌 후 npsMonthlyAmount 인풋 포커싱
  - WorkerSalary: isWorker=true 선택 → salary 인풋 포커싱
  - OtherIncome: hasOtherIncome=true 선택 → otherAnnualIncome 인풋 포커싱

### 수정 모드
- Confirm 화면에서 수정 버튼 클릭 시 해당 단계 화면으로 이동
- 이 때 버튼이 "다음"에서 "수정완료"로 변경됨
- 수정완료 클릭 시 다음 단계가 아니라 Confirm 화면으로 돌아옴

### 진입 조건 (중요)
- 55세 이하 사용자에게는 바텀시트가 표시되지 않음
- 자산동기화 실패 시에도 바텀시트가 표시되지 않음
- 바텀시트를 닫은 후에는 자산동기화 완료 화면의 기존 기능(버튼 등)이 그대로 동작해야 함

- 인출설계를 다시 시작할 경우 **Intro 화면부터** 시작

## 바텀시트 재노출 정책

- **"아니요" 선택** 후에도, 다음 자산동기화 시 조건(만 55세 초과 + 동기화 성공)이 충족되면 **바텀시트 다시 노출**
- **바텀시트가 노출되지 않는 경우**: 사용자가 설문을 **이미 제출(FA 전달 완료)**한 경우
  - 나이 조건이 충족되더라도 바텀시트 미노출
  - 제출 여부 판단은 API를 통해 확인 (미연동 상태)
