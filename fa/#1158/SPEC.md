# 계약목록 — #1158

---

## 1. 사용자 스토리

FA(금융 어드바이저)로서,
계약 중인 고객의 계약 현황을 고객 목록에서 한눈에 일괄 모니터링하고 싶다.
그래서, 각 고객 상세 페이지에 일일이 진입하지 않고도 계약목록 형태로 전체 계약 현황을 효율적으로 파악할 수 있다.

---

## 2. 배경 / 현황

### 2-1. 현재 흐름 (AS-IS)

고객목록 → 고객 클릭 → 상세 페이지 → 계약현황 탭 → 해당 고객의 계약 확인

- 계약 현황 확인 시 고객 상세 페이지에 개별 진입 필요
- 행 단위가 "고객"이라 계약 단위 모니터링 불가
- 전체 계약 포트폴리오를 한 화면에서 파악 불가

### 2-2. 변경 방향 (TO-BE)

고객관리 → **계약목록** 페이지 → 테이블 형태로 전체 계약 일괄 조회

- 행 식별 키: **계약번호 (ctrNo)**
- 한 고객이 복수 계약 보유 시 각 계약이 별도 행으로 표시
- 계약이 없는 고객은 목록에 미노출
- 해지 완료 건 목록에 미노출

## 3. 기능 요구사항

### 3-1. 화면 참고

| 영역 | 참고 화면 |
|------|-----------|
| 페이지 레이아웃 | 고객관리 > 고객목록 |
| 테이블 스타일 | 상담페이지 > 은퇴설계 > 현금흐름 |
| 데이터 구조 | 투자설계 > 투자제안 > 계약현황 |

### 3-2. 정렬 `🆕 미구현`

- 계약목록의 데이터 정렬 순서는 고객목록과 동일하게 유지
- 예: 고객목록에서 A → B → C 순이면, 계약목록에서도 A의 계약 → B의 계약 → C의 계약 순으로 표시
- 한 고객이 복수 계약 보유 시, 해당 고객의 계약끼리는 연속 배치

### 3-3. 컬럼 가시성 `🆕 미구현`

- 중요한 정보가 먼저 표출되고, 중요하지 않은 정보는 접혀 있어야 함
- 중요 컬럼 / 비중요 컬럼 분류 기준은 추후 기획 확정 필요

### 3-5. 계약상태 표시명

| 표시명 | SSI 코드 | KIS/KBS 코드 | 설명 |
|--------|----------|-------------|------|

### 3-6. UI 세부

#### 금액/수익률 색상

| 상태 | 색상 | 아이콘 |
|------|------|--------|
| 양수 (상승) | 빨강 | ↑ |
| 음수 (하락) | 파랑 | ↓ |
| 변동 없음 | 기본 텍스트 | 없음 |
| 값 없음 | gray | dash (-) |

#### 계약상태 배지

- 전체 상태 gray 톤 통일

#### 계약만료일 표시

- 30일 이하: `YYYY-MM-DD (N일)` — 일 단위, 빨강 표시
- 30일 초과 ~ 1년 미만: `YYYY-MM-DD (N개월)` — 개월 단위 (30일 기준 근사치)
- 1년 이상: `YYYY-MM-DD (N년 N개월)` — 년+개월 단위
- 만료 후: `YYYY-MM-DD (만료)` — '만료' 텍스트, 빨강 표시

> '만료' 표시가 노출되는 시점: 계약 만료일에 자동 해지 신청 후, 해지 완료 처리되기까지의 기간 동안만 노출됨.
> 해지 처리 소요 기간은 증권사별로 다름 (KBS: D+4, SSI/KIS: D+1~D+3). 정책 참고: `bw-is/해지.md`
> 해지 완료 후에는 목록에서 미노출.

#### 스코프 안내 배너 (v1)

- 테이블 상단 info 배너: "현재 KB증권 계약만 조회 가능합니다. 타 증권사 지원은 추후 순차 제공될 예정입니다."

#### 빈 화면

- 계약 데이터가 없을 경우 테이블 대신 EmptyView 컴포넌트 표시
- 21개 컬럼 테이블 특성상 테이블 내부 빈 화면은 가로 스크롤해야 보이므로, 테이블 자체를 숨기고 EmptyView로 대체

## 4. 수용 조건

- 계약목록 페이지에서 전체 계약을 계약번호 단위 행으로 조회 가능
- 해지 완료 건은 목록에 미노출
- 계약상태 배지가 gray 톤으로 일관되게 표시
- 금액/수익률 양수·음수·무변동·값없음 각 상태별 색상/아이콘 정상 표시
- v1 스코프 안내 배너가 테이블 상단에 표시
- 데이터 없을 시 테이블 대신 EmptyView 표시
