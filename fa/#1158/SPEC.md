# 계약목록 — #1158

---

## 1. 사용자 스토리

FA(금융 어드바이저)로서,
계약 중인 고객의 계약 현황을 고객 목록에서 한눈에 일괄 모니터링하고 싶다.
그래서, 각 고객 상세 페이지에 일일이 진입하지 않고도 계약목록 형태로 전체 계약 현황을 효율적으로 파악할 수 있다.

---

## 2. 기능 요구사항

### 2-1. 화면 참고

| 영역            | 참고 화면                        | 참고 이유                                                                        |
| --------------- | -------------------------------- | -------------------------------------------------------------------------------- |
| 페이지 레이아웃 | 고객관리 > 고객목록              | 고객정렬순서동일, 추후: 고객검색, 페이지네이션, 총개수                           |
| 테이블 스타일   | 상담페이지 > 은퇴설계 > 현금흐름 | 테이블 형식 표출 |
| 데이터 구조     | 투자설계 > 투자제안 > 계약현황   | 데이터 참조. 여기에 고객명만 추가.                                               |

### 2-2. 정렬

- 고객목록 기본 정렬과 일치: **고객 ID 최신순** (최신이 맨 위)
- 한 고객이 복수 계약 보유 시, 해당 고객의 계약끼리는 연속 배치

### 2-3. 화면 구성

#### 요약 카드

테이블 상단에 요약 카드 4종 표시:

| 카드 | 값 | 비고 |
|------|----|------|
| 총 계약 | 전체 건수 | |
| 운용중 | 운용 중 상태 건수 | |
| 대기중 | 대기 상태 건수 | |
| 만료 임박 (30일 이내) | 만료일 30일 이내 건수 | 빨강 강조 |

#### 상태 필터

| 필터 | 분류 기준 |
|------|-----------|
| 전체 | 모든 계약 |
| 운용중 | 상태 라벨 === '운용 중' |
| 대기중 | 라벨에 '대기' 포함 |
| 해지처리중 | 라벨에 '수수료', '해지', '출금' 포함 |
| 만료 임박 | 만료일까지 0~30일 |

### 2-4. 테이블 컬럼

테이블에는 핵심 정보 10개 컬럼을 표시하고, 상세 정보는 행 클릭 시 사이드패널에서 확인.

#### 테이블 컬럼 (10개)

| #   | 컬럼명               | 정렬  | 값                                             | 비고                               |
| --- | -------------------- | ----- | ---------------------------------------------- | ---------------------------------- |
| 1   | 고객명               | left  | 이름                                           | 왼쪽 고정 컬럼                     |
| 2   | 계약상태             | left  | 상태 표시명                                    | 배지 색상: 계약상태 배지 정책 참조 |
| 3   | 계좌유형             | left  | IRP / ISA / 연금저축 / 종합위탁                |                                    |
| 4   | 계약유형             | left  | 자문 / 일임                                    |                                    |
| 5   | 포트폴리오           | left  | 포트폴리오명                                   |                                    |
| 6   | 계약만료일           | left  | YYYY-MM-DD (N일 \| N개월 \| N년 N개월 \| 만료) | 계약만료일 표시 정책 참조          |
| 7   | 평가금액             | right | NAV (원)                                       | `formatContractAmount`             |
| 8   | 수익률               | right | %                                              | 금액/수익률 표시 정책 참조         |
| 9   | 수수료출금예정일     | left  | YYYY-MM-DD (남은일수)                          |                                    |

#### 사이드패널 (행 클릭 시)

행 클릭 시 우측 사이드패널에 상세 정보 표시:

- **헤더**: 고객명 + 계약상태 배지 + 포트폴리오명
- **성과 요약**: 평가금액, 투자원금, 수익금액, 수익률, 평가일시
- **계약 정보**: 계약유형, 계좌유형, 증권사, 운용계좌, 계약기간, 계약시작일, 계약만료일, 남은 계약 기간
- **수수료 정보**: 수수료 유형, 수수료율, 출금 주기, 출금 기관, 출금 계좌, 출금 예정일

## 3. 수용 조건

- [ ] 계약목록 페이지에서 전체 계약을 계약번호 단위 행으로 조회 가능
- [ ] 계약이 없는 고객은 목록에 미노출
- [ ] 해지완료(2130/2290) 건은 목록에 미노출
- [ ] 목록 정렬: 고객 ID 최신순, 동일 고객의 복수 계약은 연속 배치
- [ ] 요약 카드: 총 계약, 운용중, 대기중, 만료 임박(30일 이내) 건수 표시
- [ ] 상태 필터: 전체, 운용중, 대기중, 해지처리중, 만료 임박 필터 동작
- [ ] 계약상태 배지: 운용 중=초록, 해지 진행 중(2231/2234/2235)=빨강, 그 외=노랑
- [ ] 수익금액: 양수 빨강(▲ 아이콘), 음수 파랑(▼ 아이콘), 0 → `0원`, 값없음 → `-`
- [ ] 수익률: 양수 빨강(`+N.NN%`), 음수 파랑(`-N.NN%`), 0 → `0.00%`, 값없음 → `-`
- [ ] 계약만료일: 30일 이하·만료 후 빨강 표시, 기간별 단위(일/개월/년+개월/만료) 전환
- [ ] 행 클릭 시 사이드패널에 계약 상세 정보(성과/계약/수수료) 표시
- [ ] 가이드 문구를 테이블 상단에 표시
- [ ] 데이터 없을 시 테이블 대신 EmptyView 표시, 필터 결과 없을 시 안내 메시지 표시
